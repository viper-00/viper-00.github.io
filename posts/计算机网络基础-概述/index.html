计算机网络基础 🔗

概述。

ISP 🔗

互联网服务提供商 （Internet service provider）从互联网管理机构获得 IP 地址，并通过管理和分发帮助个人和机构接入互联网服务，例如邮件服务、域名注册、web hosting 和浏览包等。一般只需要交少许钱就可以享受互联网服务。

计算机网络体系结构 🔗

OSI：应用层、表示层、会话层、运输层、网络层、数据链路层、物理层。 五层协议：应用层、运输层、网络层、数据链路层、物理层。 TCP/IP：应用层、运输层、网际层、网络接口层。

五层协议可以作为 OSI 和 TCP/IP 的整体，所以一般我们探讨计算机的网络结构都以五层协议为基础，理解每层的意义和作用。

  • 应用层：为操作系统和应用程序提供访问网络服务的接口，例如 FTP（文件传输协议）、HTTP 协议、DNS（域名解析协议）、Telnet（远程登录协议）和 SMTP（邮件传送协议）。数据传输的基本单位是报文。
  • 传输层：为上层数据分段并提供端到端、可靠或不可靠的传输，主要包括传输控制协议 TCP（Transmission Control Protocol）和用户数据报协议 UDP（User Datagram Protocol）。其中 TCP 主要提供完整性服务，数据单位为报文段，而 UDP 主要提供及时性服务，数据单位为用户数据报。
  • 网络层：为主机提供数据传输服务，做路由选择。网络层把传输层传递下来的报文段或用户数据报封装成分组。基本数据单位为 IP 数据报。
  • 数据链路层：根据链路层协议为同一链路的主机提供可靠的数据传输服务。基本数据单位为帧。
  • 物理层：为上层协议提供可靠的数据传输媒体，硬件上，确保原始数据转化为比特流能在物理媒体上传输。基本数据单位是比特流。

物理层 🔗

第五层协议。

通信方式 🔗

根据信息的传输方式可分为如下三种通信方式：

  • 单工通信：单向传输
  • 半双工通信：双向交替传输
  • 全双工通信：双向同时传输

数据链路层 🔗

第四层协议。

信道分类 🔗

  • 广播信道：一对多通信。一个节点发送的数据能够被广播信道上的所有节点接收。因为容易发生碰撞和冲突，所以需要协调，如利用信道复用技术和 CSMA/CD 协议。
  • 点对点信道：一对一通信。不发生碰撞，用 PPP 协议实现。一般互联网用户与 ISP 通信的链路层协议使用 PPP 协议。

MAC 地址 🔗

MAC 地址是链路层地址，长度为 6 字节（48位），用于标识网络适配器（网卡）。

一台计算机的 MAC 地址数量与网络适配器数量有关，例如一般的笔记本电脑拥有无线和有线两种网络适配器，所以应该有两个 MAC 地址。

局域网 🔗

局域网是一种广播信道，是一种私有网络，主要特点是为单位所有，比如家庭、办公室和工厂，地理范围和站点数目均有限。

按照网络拓扑结构对局域网进行分类主要有三种：星型、环型和直线型。

以太网 🔗

以太网是一种星型拓扑结构的局域网，是一种全球化的电线电缆系统标准，将世界各地的计算机、设备和机器连接在一起。早起以太网使用集线器连接，而现在大量采用交换机。交换机能根据 MAC 地址转发和存储数据。

交换机 🔗

交换机是一种用于电（光）信号转化的网络设备，利用内部的交换表存储的 MAC 地址寻找对应的接口映射，实现数据交换。最常见的交换机是以太网交换机。

网络层 🔗

第三层协议。

网络层是整个互联网的核心，使用 IP 协议把所有异构的物理网路连接起来，看成是一个整体。

与 IP 协议配套使用的还有三个协议：

  • 地址解析协议 ARP（Address Resolution Protocol）
  • 互联网控制消息协议 ICMP（Internet Control Message Protocol）
  • 互联网组管理协议 IGMP（Internet Group Management Protocol）

地址解析协议 ARP 🔗

地址解析协议是一个通过解析网络层地址来寻找数据链路层地址的网络传输协议，在 IPv4 中应用广泛。

ARP 实现由 IP 地址得到数据链路层的 MAC 地址。

互联网制消息协议 ICMP 🔗

互联网控制消息协议用于在网际协议（IP）中发送控制消息，在通信环境中发现各种问题，并给予反馈。

PING 🔗

Ping 是 ICMP 中的一个重要应用，主要用来测试两台主机之间的连通性。

Ping 的原理是向目的主机发送 ICMP Echo 请求报文，目的主机收到并响应后，对源主机发送 Echo 回答报文，此时双方主机通信成功。

Ping 会根据响应时间和成功率估算出数据包的往返时间和丢包率。

互联网组管理协议 IGMP 🔗

互联网组管理协议是用于管理网络协议多播组成员的通信协议。

IP 主机利用 IGMP 将主机的多点发送成员人数报告发送给临近的多点发送路由器。

传输层 🔗

第二次协议。

网络层只把分组发送到目的主机，而真正通信的不是主机而是主机中的进程。传输层提供了进程间的通信逻辑，屏蔽了下面网络层的核心细节，使程序看起来像是在两个传输层实体之间有一条端到端的逻辑通信信道。

UDP（User Datagram Protocol） 🔗

用户数据报协议 UDP 是无连接的，尽最大可能交付，没有拥塞控制，面向报文，支持一对一、一对多、多对一和多对多的交互通信。

TCP（Transmission Control Protocol） 🔗

传输控制协议 TCP 是有连接的，提供可靠交互，有流量控制、拥塞控制，提供全双工通信，面向字节流，每一条 TCP 连接只能是点对点（一对一）的交互通信。

TCP 的三次握手 🔗

假设 A 是客户端，B 是服务器端。

  • 开始，B 处于 LISTEN（监听）状态，等待客户端的连接请求。
  • A 向 B 发送连接请求报文。
  • B 收到连接请求报文，如果同意建立连接，那么向 A 发送连接确认报文。
  • A 收到 B 的连接确认报文后，再向 B 发送返回确认。
  • B 收到 A 的确认，此时，双方的 TCP 连接建立成功。

第三次握手的原因 🔗

第三次握手是为了防止失效的连接请求到达服务器，让服务器错误打开连接。

第一次握手的过程中，如果客户端发送的连接请求在网络中滞留，出现超时，那么此时客户端就会因为超时重传机制，重新发送请求。此时前一个请求和新发的请求都会到达服务器端。如果没有第三次握手，服务器就会打开两个 TCP 连接，与客户端发送的两次请求建立连接。如果有第三次握手，当服务端返回前一个请求的确认后，客户端就会忽略这一请求，因为客户端知道这条请求连接已经重发过。

所以，第三次握手是为了防止建立不必要的连接。

TCP 的四次挥手 🔗

假设 A 是客户端，B 是服务器端。

  • A 发送连接释放报文。
  • B 收到之后发出确认，此时 TCP 属于半关闭状态，B 能向 A 发送数据但 A 不能向 B 发送数据。
  • 当 B 不再需要连接时，向 A 发送连接释放报文。
  • A 收到后发出确认，等待 2 MSL（最大报文存活时间）后释放连接。
  • B 收到 A 的确认后释放连接。

TCP 可靠传输 🔗

TCP 使用超时重传机制来实现可靠传输。如果一个已经发送的报文段在超时时间内没有收到确认，那么就重新发送这个报文段。

应用层 🔗

第一层协议。

域名解析协议（DNS） 🔗

常用端口：53，传输层协议：UDP/TCP。

DNS 是一个分布式数据库，提供主机名和 IP 地址之间相互转化的服务。分布式数据库是指每个站点只保留自己那部分数据。分布式系统保证通过网络通信来使各个节点共同完成任务。

域名具有层次结构，从上到下依次为：根域名（Root Domain）、顶级域名（Top-level Domains）、二次域名（Second-level Domains）。

  • 根域名："."
  • 顶级域名：“edu”, “gov”, “com”, “net”
  • 二级域名：“google”, “baidu”, “github”, “mit”

大多数情况下使用 UDP 进行传输，所以就要求域名解析器和域名服务器来处理超时和重传，从而保证连接的可靠性。

文件传输协议（FTP） 🔗

常用端口：20/21，传输层协议：TCP。

FTP 使用 TCP 进行连接，需要建立两个连接来传送一个文件：

  • 控制连接：服务器端开放 21 端口等待客户端的连接，客户端发送连接后，等待服务端的响应和应答请求。双方都确认后，建立连接成功。
  • 数据连接：服务器端开放 20 端口，等待客户端传送的文件数据。

远程登录协议（Telnet） 🔗

常用端口：23，传输层协议：TCP。

Telnet 用来连接登录到远程主机，实现面向文本的双向交互。所以，远程主机上的输出内容会返回回来。

随着互联网的发展，SSH（Secure Shell）协议逐渐取代了 Telnet 的应用范围。

邮件传送协议（SMTP） 🔗

一个电子邮件系统由三部分组成：用户代理、邮件服务器和邮件协议。

邮件协议包含发送协议和读取协议，发送协议通常用 SMTP，读取协议通常用 POP3 和 IMAP。SMTP 使用 TCP 进行连接。

Simple Mail Transfer Protocol 🔗

常用端口：25，传输层协议：TCP。

SMTP 只能发送 ASCII（American Standard Code for Information Interchange）码。因为互联网邮件内容的多样化，普通的文件格式已经无法满足普通用户的需求。所以，在 SMTP 的基础上增加了 MIME（multipurpose Internet Mail Extensions）扩展格式，从而支持音频、视频、图像和各种应用程序的二进制文件传输。

Post Office Protocol 3 🔗

常用端口：110，传输层协议：TCP。

POP3 协议的特点是只要用户从邮件服务器上读取了邮件，就立马将邮件删除，不留备份。

Internet Message Access Protocol 🔗

常用端口：143，传输层协议：TCP。

IMAP 协议的特点是保持客户端和服务器上的邮件同步，用户不手动删除邮件，那么服务器也不会删除。IMAP 协议给予用户更多的自主权。
