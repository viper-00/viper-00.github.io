计算机操作系统 🔗

概述。

基本特征 🔗

并发/并行 🔗

  • 并发：指宏观上在一段时间内能同时运行多个程序。
  • 并行：同一时刻能运行多个程序。

实现并行需要硬件的支持，如多流水线、多核处理器、分布式操作系统。

操作系统利用进程和线程实现程序的并发运行。

共享 🔗

共享指系统中的资源可以被多个并发程序共同使用。

  • 互斥共享：资源为临界资源，同一时刻只允许一个进程访问，需要使用同步机制来解决互斥问题。比如打印机。
  • 同时共享：同一时刻允许多个进程访问，一般情况下是只读文件。

虚拟 🔗

虚拟技术就是以单个物理硬件系统为基础创建多个模拟环境或专用资源，通过组合和分区把现有的计算机资源抽象分割成多个电脑配置环境。

复用技术：

  • 时分复用技术：多进程使用同一处理器实现了时分复用技术，每个进程都轮流占用处理器资源，且每次执行小个时间片段，并快速切换上下文交替进行。
  • 空分复用技术：虚拟化内存使用了空分复用技术。将物理内存抽象成地址空间，每个进程都有各自的地址空间。地址空间的页被映射到物理内存，当使用到一个没有在物理内存的页时，执行页面置换算法，将该页置换到内存中。

异步 🔗

进程异步执行可以避免主进程运行过程中发生阻塞，把有延迟的请求给异步做，当处理完成后再通知主进程继续完成任务。

系统调用 🔗

计算机中，进程在用户态调用内核态的功能需要通过系统调用接口（system call interface），转而让操作系统完成指定操作。

Linux 操作系统的系统调用：

任务 指令/命令
进程控制 fork(); exit(); wait();
进程通信 pipe(); shmget(); mmap();
文件操作 open(); read(); write();
设备操作 iotl(); read(); write();
信息维护 getpid(); alarm(); sleep();
安全 chmod(); umask(); chown();

内核 🔗

操纵系统内核的设计架构一般有三种：宏内核、微内核和混合内核。不同的设计架构有不同的功能和应用场景。

宏内核 🔗

宏内核又称集成式内核、单体式内核，是将操作系统的核心功能都集成到内核中，所以各个模块之间的通信效率很高，有很高的性能。

微内核 🔗

由于计算机技术的发展，操作系统内部功能不断复杂，因此将里面的一部分功能移出内核，能大大降低内核的复杂性。分出的模块各自管理、相互独立。此时的内核被称为微内核。

微内核的内核态存放基本模块，例如 CPU 调度、内存管理和进程间通信等，而其他模块都在用户态，如文件系统、设备驱动和应用程序等。

混合内核 🔗

混合内核是在宏内核和微内核的基础上改进的一种内核设计架构。在 Linus 看来这是一种市场营销手段，因为混合内核的实现接近于宏内核。

中断分类 🔗

外中断 🔗

CPU 执行以外方式引发，例如 I/O 中断、时钟中断、控制台中断。

异常 🔗

CPU 执行指令后内部引发的错误，例如非法操作码、地址越界、算术溢出等。

陷入 🔗

用户程序中使用系统调用。

进程管理 🔗

进程 🔗

进程是操作系统资源分配的基本单位。

进程控制块（Process Control Block，PCB）是一种数据结构，在计算机操作系统中用来存储所有进程信息。当一个进程创建时，操作系统会同时创建 PCB，PCB 中保存着进程的描述符等基本信息和运行状态。

线程 🔗

线程是操作系统能够运行独立运算调度的最小单位。大多数情况下是在进程下运行，是进程的实际运作单位。

一个进程可以有多个线程，这些线程共享着进程资源。

微信和浏览器是两个应用程序，运行着两个进程。浏览器内有很多线程在运作，例如 HTTP 请求线程、渲染线程、响应线程等。线程实现并发可以让用户同时做多件事情，例如点击链接请求数据的同时浏览其他页面。

进程和线程的区别 🔗

  1. 拥有资源

进程是资源分配的基本单元，而线程没有资源。线程可以使用和访问隶属进程的资源。

  1. 调度

线程是独立运算调度的最小单位，同一进程中，线程的切换不会引起进程的切换，从一个进程中的线程切换到另一个进程中的线程时，会引起进程切换。

  1. 系统开销

在创建和销毁（分配和回收）系统资源时，进程比线程需要更多的开销。进程间切换涉及当前执行进程 CPU 环境的保护和新调度进程 CPU 环境的设置。线程间切换只需保存和设置少量寄存器内容，所以开销很少。

  1. 通信

进程间通信利用 IPC（Inter-process communication）。线程间通过读取同一进程中的数据进行通信。

进程状态 🔗

  • 创建进程（crated）：创建进程
  • 就绪状态（ready）：等待被调度
  • 运行状态（running）：正在运行中
  • 阻塞状态（waiting）：等待资源分配
  • 终止状态（terminated）：进程结束终止

只有就绪状态和运行状态可以相互切换，其他都是单向。进程在就绪状态通过调度算法获得 CPU 时间后，转化为运行状态。当在时间片段用过之后就会转为就绪状态，等待下一次被调度。

进程在运行状态缺少系统资源后会转变成阻塞状态，这时的 CPU 时间还未使用完，但进程停滞不前，所以重置时间并回到就绪状态，并再次等待 CPU 资源分配。

当一个进程在运行状态结束后，会退出转变成终止状态。

进程调度算法 🔗

不同环境的调度算法不同，一般有这几种环境：批处理系统、交互式系统、实时系统。

批处理系统 🔗

批处理系统的调度算法能保证吞吐量和周转时间（创建到终止的时间）。

  1. 先来先服务 first-come first-serverd（FCFS）

按照请求的顺序进行调度。有利于长作业，不利于短作业。

  1. 短作业优先 shortest job first（SJF）

计算运行时间最短的顺序进行调度。有利于短作业，如果长作业工作时一直有短作业存在，那么长作业会一直得不到调度。

  1. 最短剩余时间优先 shortest remaining time next（SRTN）

计算剩余时间的顺序进行调度。当一个新的作业到达时，与当前作业的剩余时间做比较，如果新作业的时间更短，那么挂起当前作业，运行新作业，否则继续当前作业。

交互式系统 🔗

交互式系统有大量的用户交互操作，所以系统调度算法的目标是快速响应用户请求。

  1. 时间片轮转

所有进程在就绪状态按 FCFS 原则排成一个队列，调度时把 CPU 时间分配给队首进程。当进程的时间片用完时，计时器发出时钟中断，调度程序立即停止该进程执行，并将该进程送往就绪队列的末尾。以此往复，继续执行队首的进程。

时间片轮转有两个问题：

  • 如果进程分配的时间片太小，会导致进程频繁切换，进程在切换的过程中需要保存进程的信息和载入新进程的信息，所以最终系统的资源会大量浪费。
  • 如果进程分配的时间片太长，那么其他进程的实时性就无法保证。
  1. 优先级调度

为每个进程分配一个优先级，按优先级高低进行调度。

为了防止低优先级进程得不到执行，可以在一定时间后修改这些进程的优先级大小。

  1. 多级反馈队列

多级反馈队列设置了多个队列，每个队列的时间片大小不同。进程在第一个队列没执行完就会被移到下一个队列。每个队列的优先级不同，最上面的优先级最高，从上往下按顺序执行。

可以将这种调度算法看作是时间片轮转调度算法和优先级调度算法的结合体。

实时系统 🔗

实时系统要求用户的请求在一定时间内完成。

系统的调度算法应该最先处理这时发送的进程请求。

进程同步 🔗

临界区 🔗

对临界资源进行访问的那段代码称为临界区。

为了防止访问临界资源时进程之间发生互斥，所以在进入临界区之前，要先对进程进行检查。

同步和互斥 🔗

  • 同步：多个进程通过合作相互联系，进程间有先后执行关系。
  • 互斥：多个进程在同一时刻只能有一个进入临界区。

信号量 🔗

信号量（Semaphore）又称为信号标，是一个同步对象，用来保持在0到指定最大值之间的一个计数值。信号量有 up 和 down 两种操作。P 代表通过，V 代表释放。

  • down：当信号量大于 0，执行 -1 操作；当信号量等于 0，进程睡眠。
  • up：对信号量执行 +1 操作，唤醒睡眠的进程让其工作。

进程通信 🔗

进程同步和进程通信很容易混淆，它们的区别在于：

  • 进程同步：控制多个进程并按一定顺序执行。
  • 进程通信：进程间传输信息。

进程通信是一个手段，而进程同步是一种目的。为了能让进程同步，需要让进程进行通信，传输一些进程同步所需要的内容/信息。

管道 🔗

操作系统进程的管道由 pipe 函数实现。

管道限制：

  • 只支持半双工通信（单向交替传输）。
  • 只能在父子进程或者兄弟进程之间使用。

FIFO 🔗

命名管道，能在父子进程以外的场景下通信。

FIFO 常用于客户端-服务器程序中，先进的请求先处理。

消息队列 🔗

略。

信号量 🔗

计数器，为多个进程提供共享数据对象的访问。

共享存储 🔗

允许多个进程共享一个给定的存储区，进程之间不需要复制数据，所以效率更高。

使用信号量来控制进程对存储区的访问。

多个进程可以将同一个文件映射到指定的地址空间从而实现共享内容。

套接字 🔗

用于在不同机器之间的进程通信。

死锁 🔗

死锁（deadlock）又称为死结。当有两个以上的运算单元，双方都在等待对方停止执行并获取系统资源，且没有一方提出退出时，就会发生死锁。

发生死锁的四个条件：

  • 禁止抢占（no preemption）：系统资源不能被强制从一个进程中退出。
  • 持有和等待（hold and wait）：一个进程可以在等待时持有系统资源。
  • 互斥（mutual exclusion）：资源只能同时分配给一个进程，无法多个进程共享。
  • 循环等待（circular waiting）：一系列进程互相持有其他进程所需要的资源。

因为系统解决死锁的代价很高，因此大多数操作系统采用鸵鸟策略（忽略问题）来对待死锁。

设备管理 🔗

磁盘结构 🔗

  • 盘面（Platter）：一个磁盘有多个盘面。
  • 磁道（Track）：盘面上的圆形带状区域，一个盘面可以有多个磁道。
  • 扇区（Track Sector）：磁道上的一个弧段，一个磁道可以有多个扇区，它是最小的物理存储单位，目前主要有 512 bytes 和 4k 两种。
  • 磁头（Head）：与磁面非常接近，能够将盘面上的磁场转换为电信号（读），或者将电信号转化为盘面的磁场（写）。
  • 制动手臂（Actuator arm）：用于在磁道之间移动磁头。
  • 主轴（Spindle）：使整个盘面转动。

磁盘调度算法 🔗

读写一个磁盘块的时间的影响因素：

  • 旋转时间（主轴转动盘面，使磁头移动道适当的扇区上）
  • 寻道时间（制作手臂移动，使磁头移动到适当的磁道上）
  • 实际的数据传输时间

先来先服务（First Come First Served，FCFS） 🔗

按照请求的顺序进行调度，不对磁盘寻道做任何优化。

最短寻道时间优先（Shortest Seek Time First，SSTF） 🔗

优先调度与当前磁头距离最近的磁道。

电梯算法/扫描算法（SCAN） 🔗

电梯算法保持一个方向运行，直到该方向没有请求，才改变运行方向。
